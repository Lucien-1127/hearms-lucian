# 智研 AI — 法律 AI 系統深度研究報告

> 研究日期：2026-07-05

## 一、台灣法律 AI 系統現況分析

### 1.1 市場現有系統

| 系統 | 特點 | 限制 |
|------|------|------|
| TaiLexi AI | 台灣法律 GPT，訓練於法院判決與法條 | 閉源，無 API |
| Lawbot AI | 法律文件搜尋、書狀編輯 | 聚焦文件生成，不含 RAG |
| LexiHK | 香港法律 LLM（HKGAI 基礎） | 不適用台灣法律 |
| 司法院 e 服務 | 官方查詢系統 | 無 AI 輔助，操作繁瑣 |

### 1.2 核心技術缺口

1. **RAG 整合不足**：多數系統直接對 LLM 提問，缺乏真實法條/判例的即時檢索
2. **多輪對話能力弱**：不支援複雜法律問題的深度追問
3. **文件分析功能缺失**：少有系統能自動識別合約風險條款
4. **開源生態薄弱**：台灣法律 AI 開源資源極為匱乏

## 二、智研 AI 系統設計原則

### 2.1 核心設計理念

```
準確性 > 速度 > 覆蓋度
```

- **準確性優先**：每個答案必須引用具體法條/判例，不允許「幻覺」
- **透明度**：明確標示資訊來源與時效限制
- **安全邊界**：明確區分「法律資訊」與「法律意見」

### 2.2 RAG 架構設計

└──────────────┬───────────────┘
               │

### 2.3 資料來源規劃

| 資料集 | 內容 | 更新頻率 | 取得方式 |
|--------|------|---------|---------|
| 全國法規資料庫 | 所有現行法規 | 即時 | 官方 OpenData API |

| 大法官解釋 | 憲法解釋 | 不定期 | 司法院官網 |

## 三、系統架構深度分析

### 3.1 後端服務分解

```python
# 三個核心路由
/api/v1/consultation  → AI 法律諮詢（多輪對話 + RAG）
/api/v1/search        → 語意判例搜尋（ChromaDB）
/api/v1/documents/analyze → 文件風險分析
```

### 3.2 AI Prompt 工程策略

**法律諮詢 System Prompt 設計原則：**

1. **角色定義**：明確定位為「研究助理」而非「律師」
2. **輸出結構化**：強制 JSON 格式確保 references 可解析
3. **溫度控制**：低溫（0.3）確保法律內容一致性
4. **Token 預算**：4096 tokens，允許詳細引用

**文件分析 System Prompt 設計原則：**

1. **風險三級制**：低/中/高風險，對應條款具體化
2. **修改建議**：每個問題都附帶可執行的修改方向
3. **整體評估**：最終整合為「整體風險等級」

### 3.3 向量嵌入選型

| 模型 | 中文支援 | 維度 | 速度 |
|------|---------|------|------|
| `sentence-transformers/paraphrase-multilingual-mpnet-base-v2` | ✅ 優 | 768 | 中 |
| `BAAI/bge-m3` | ✅ 極佳 | 1024 | 慢 |
| `text-embedding-3-small` (OpenAI) | ✅ 良 | 1536 | 快（API） |

**推薦**：開發階段使用 `paraphrase-multilingual-mpnet-base-v2`，生產環境升級至 `BAAI/bge-m3`。

## 四、安全性與合規考量

### 4.1 資料隱私

- 用戶問題不儲存至資料庫（無狀態 API）
- 文件分析後立即釋放記憶體，不留存用戶文件
- ANTHROPIC_API_KEY 僅於伺服器端使用，前端不暴露

### 4.2 法律合規

- 明確標示系統為「資訊參考」而非「法律意見」
- 系統輸出附帶免責聲明
- 不處理個資敏感案件（如涉及真實當事人姓名的刑事案件）

### 4.3 AI 倫理邊界

```
允許                    不允許
────────────────────    ───────────────────
✅ 解釋法條含義          ❌ 給出具體訴訟策略
✅ 列舉相關判例          ❌ 預測個案勝訴率
✅ 說明法律風險          ❌ 代替律師出具意見書
✅ 合約條款分析          ❌ 涉及機密訴訟資訊
```

## 五、效能與擴展規劃

### 5.1 當前系統限制

| 指標 | 當前 | 目標 |
|------|------|------|
| 並發請求 | 10 req/s | 100 req/s |
| 法規資料量 | 0（初始） | 500+ 法規 |
| 判例資料量 | 0（初始） | 50萬+ 判決 |
| 回應時間 | ~3-5s | <2s |

### 5.2 擴展路線圖

**Phase 1（MVP）**：當前已實作
- AI 法律諮詢（Claude API）
- 文件分析
- 判例語意搜尋（AI 生成）

**Phase 2（資料填充）**：
- 接入全國法規資料庫 API
- 爬取司法院裁判書並向量化
- 建立法律知識圖譜

**Phase 3（進階功能）**：
- 訴狀草稿生成
- 法律費用試算
- 律師媒合推薦
- 多語言支援（英文、閩南語）

## 七、結論

智研 AI 定位為**台灣首個全功能開源法律 AI 系統**，核心差異化在於：

1. **開放架構**：可自架、可二次開發
2. **RAG 優先**：確保回答有法條依據
3. **完整功能**：諮詢 + 搜尋 + 文件分析三合一
4. **企業就緒**：Docker 部署、REST API、可接入既有系統
